迷路を作成するためのアルゴリズムには、いくつかの主要な手法があります。これらのアルゴリズムは、生成される迷路の特性（複雑さ、道順の一意性など）に違いがあります。

代表的な迷路生成アルゴリズムには、以下のようなものがあります。

## 迷路生成アルゴリズムの主な種類

### 1. 穴掘り法（Depth-First Search, DFS ベース）
「壁を掘って道を作っていく」という考え方に基づいています。

* **特徴:**
    * **深さ優先探索**（DFS）の考え方を利用しており、通常、再帰的に実装されます。
    * すべてのセルが壁で埋まった初期状態から始め、ランダムな方向を選んで進み、**未訪問のセル**であればその間の壁を壊して通路にします。
    * 行き止まりになったら、**一つ前のセルに戻り**（バックトラック）、別の方向への探索を試みます。
    * **一筆書き**のような、枝分かれが少なく、長い通路（単一の解を持つ）を持つ迷路になりやすいです。

### 2. 壁伸ばし法（Wall Following / Growing Tree Method）
通路から壁を伸ばしていく、あるいはランダムな点から壁を成長させていく手法です。

* **特徴:**
    * 初期状態をすべて通路として、ランダムな地点から壁を伸ばし、既存の壁に接続させます。
    * 棒倒し法と似ている部分もありますが、より複雑な迷路が生成される傾向があります。

### 3. 棒倒し法（Aldous-Broder Algorithm や簡略化されたもの）
主にシンプルな迷路生成に使われます。

* **特徴:**
    * 格子状に配置された「柱」を起点とし、その柱をランダムに選んだ四方（上下左右）のいずれかの方向に「倒す」（通路と壁のパターンを作る）ことで迷路を作ります。
    * 比較的**実装が容易**ですが、迷路の入口付近の道が単純になりやすいという傾向があります。

### 4. プリム法 / クラスカル法（Prim's / Kruskal's Algorithm ベース）
**最小全域木**（Minimum Spanning Tree, MST）を求めるアルゴリズムを応用した手法です。

* **プリム法:**
    * 一つのセルから始め、その周辺の壁をリスト化し、ランダムに選んだ壁を壊して迷路を広げていきます。
    * **「広がりのある自然な形状」**の迷路になりやすいです。

* **クラスカル法:**
    * すべての壁をリスト化し、ランダムな順番で壁を壊していくことで通路を作りますが、このとき、壁を挟んだ両側のセルが既に繋がっていないことを確認する必要があります（**ループを作らない**）。
    * **「網目状の迷路」**になりやすいです。

### 5. 再帰的分割法（Recursive Division）
迷路全体を壁で区切り、その壁にランダムに穴を開けていくことを**再帰的に繰り返す**手法です。

* **特徴:**
    * 迷路を長方形に分割し、それぞれの長方形の内部に壁を作り、その壁に必ず**一つの通路（穴）**を開けます。
    * 部屋のような構造を持つ迷路や、比較的**直線的な通路**が多く、整然とした迷路になりやすいです。

これらのアルゴリズムは、プログラミングによって実装され、それぞれの特性に応じたユニークな迷路を生成します。
